# CONTEXT.md format

The working vocabulary and boundaries for one talk — pinned before and during outlining so the material, and a series it belongs to, doesn't sprawl. A written, shared frame beats re-deciding the same terms in chat every session.

Lives at `talks/<NN-slug>/CONTEXT.md`, one per talk. Framing decisions made while drafting go here, not only into the conversation.

## Structure

```md
# <Talk title>

<One line: what this CONTEXT is for — which talk, where it sits.>

## Language
**<Term>**:
<Definition as used in THIS talk.>
_Avoid_: <what this term does NOT mean here; the neighboring meaning to keep out>

## Relationships
- <how terms and parts connect; how this talk borders its neighbors in a series>

## Example dialogue
- Q: <a recurring question about the frame>
  A: <how you answer it, in the frame's own terms>

## Flagged ambiguities
- <an ambiguity you hit and how you resolved it — so it stays resolved>
```

## Rules

- **Define terms against their boundaries.** Every term carries an *Avoid* line — the neighboring meaning you keep out is half the definition.
- **Series borders live here.** What a neighboring talk covers is a boundary, not a repeat — write it down so the outline honors it.
- **Resolve once.** A decision about the frame goes into *Flagged ambiguities*; don't re-litigate it next session.
- **One talk per file.** The frame is the talk's; the speaker's voice is the profile's. Keep them apart.
