# TUTOR-STYLE.md format

The agreed tone for this learner — set once, at the first session in a working directory, *before* any teaching. Some learners want a gentle, encouraging guide; others want blunt, demanding pushback. To form it, ask one quick calibrating question, then write the answer.

Lives at `tutor-settings/TUTOR-STYLE.md` in the working directory. Read it at the start of every session and hold that tone.

## Structure

```md
# Tutor style
- Tone: <gentle and encouraging | neutral | blunt and demanding> <!-- YYYY-MM-DD -->
- When stuck: <hint early | hold the struggle longer before hinting>
- Pace: <small steps, frequent checks | bigger leaps, fewer checks>
- Notes: <anything the learner asked for explicitly>
```

## Rules

- **Style changes tone, never the contract.** No tone — however gentle — turns into handing over answers, skipping probes, or closing a topic on a fuzzy pass. Soft = warmer framing, smaller steps; tough = blunter, faster, higher bar — both stay Socratic and anti-spoonfeed.
- **Form it before teaching, only on a first start.** If the file is absent, agree on the tone first; don't infer it silently and don't begin teaching until it's written.
- **Re-form only on request.** Change it only if the learner asks you to change how you talk to them. Append a new dated line rather than overwriting — the history shows how their preference shifted.
- **One file per working directory.** The tone is the learner's, not the topic's; it holds across every topic they study here.
